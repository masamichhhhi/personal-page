# Research: Blog Layout Revamp

## 1. デザイン参照の取り込み方針

- **Decision**: 参照ページの雰囲気(配色、タイポグラフィ、余白、カード構成、視覚的階層)をトップページ全体に反映する。
- **Rationale**: 参照先の情報設計が明瞭で、視認性と回遊性を高めやすい。
- **Alternatives considered**: 既存デザインの軽微調整のみ(印象が変わりにくい)、一部セクションのみ変更(統一感が弱い)。

## 2. トレンド機能の扱い

- **Decision**: トレンド表示は実装しない。
- **Rationale**: 要件で不要と明言されており、データの解釈や更新基準が不明なため。
- **Alternatives considered**: 期間指定の閲覧数ベースなどを導入(仕様の追加が必要になる)。

## 3. データ構造と取得

- **Decision**: 既存のコンテンツ構造を維持し、レイアウト側で配置と見た目を再構成する。
- **Rationale**: 既存の投稿データに依存する前提であり、データ変更は要件外。
- **Alternatives considered**: 新しいフィールドや分類の追加(仕様拡張が必要になる)。

## 4. 技術スタックの変更

- **Decision**: Astro + Tailwind を維持し、既存のフロントエンド構成の範囲で改修する。
- **Rationale**: 既存プロジェクトの依存関係を活かし、開発・保守コストを最小化できる。
- **Alternatives considered**: UIフレームワークの導入、CSS構成の全面変更(工数増)。
